#### 1.1 Обоснование выбранной темы

- Интернет является неотъемлемой частью жизни современного человека. С увеличением числа пользователей, требования к качеству и функционалу web-приложений возрастают, что приводит к появлению большого числа фреймворков на рынке, которые значительно упрощают процесс разработки. Разработчики стремятся к тому, чтобы фреймворки обладали гибкостью, масштабируемостью, легкостью.
За последние годы было создано более 500 различных языков программирования и инструментов, но не каждый из них отвечает таким требованиям, как эффективность и универсальность. 
Выбор данной темы исследования обусловлен потребностью актуализировать преимущества и недостатки наиболее востребованных фреймворков в сфере HTML (Django, FastAPI, Flask), а также, апробировать, какой из них на языке Python более практичен и при этом лёгок в понимании и написании.

#### 1.4 Основные определения

1) Веб-приложение (web-приложение) - Веб приложение- способ хранения и передачи информации пользователю, а также предоставления широкого спектра услуг в онлайн режиме. 
2) Фреймворк («framework» - каркас) - программная заготовка, шаблон, позволяющий ускорить и упростить решение типовых задач. 
3) Сайт-каталог - разновидность корпоративных сайтов, промежуточный вариант между стандартным имиджевым порталом и полноценным интернет-магазином. 
4) Маршрутизация – процесс определения действий, которые должны выполнятся при обращении к определенному URL-адресу.
5) Аутентификацией – процесс идентификации пользователя, позволяющий установить его право доступа к определенной информации или ресурсам. 
6) Объектно реляционное отображение (ORM - Object Relational Mapping) -  метод, используемый для создания «моста» между реляционными базами данных и объектно-ориентированными программами.
7) Интерфейс прикладного программирования (API - Application Programming Interface) - код, обеспечивающий передачу данных между одним программным продуктом и другим.

### 2. Свойства и особенности фреймворков
----
Фреймворк предоставляет разработчикам структуру и стандарты, которые помогают сосредоточиться на логике приложения, а не на однотипных задачах.
В основном в состав фреймворка входят готовые модули для работы с базами данных, маршрутизацией, аутентификацией и другими важными аспектами веб-разработки. Например, вместо того чтобы каждый раз писать код для подключения к базе данных, фреймворк предоставляет готовые решения, которые можно использовать повторно. Это не только экономит время, но и снижает вероятность ошибок.
Преимущества использования фреймворков:
- высокая скорость разработки, которая достигается за счет открытого доступа к документации и множества форумов
- использование встроенных шаблонов помогает создавать проекты более высокого качества, в которых не сложно ориентироваться
- фреймворки являются полностью бесплатными и имеют открытый исходный код
Но каждый фреймворк индивидуален и имеет собственный ряд преимуществ и недостатков. Рассмотрим их на конкретных примерах.

#### 2.1 Django

Django - высокоуровневый бесплатный фреймворк, созданный в 2003 году, с его помощью можно быстрее и проще реализовывать сайты и веб-приложения на Python, которые работают в браузере. 
В архитектурном плане Django работает по принципу MVT (Model-View-Template), что позволяет разделить логику приложения на три части:
Model - модель, отвечающая за структуру и поведение данных;
View - представление, обрабатывающая логику отображения данных;
Template - шаблон, отвечающий за отображение пользовательского интерфейса.
Основные особенности Django:
- Бесплатно и с открытым исходным кодом: так как Django является бесплатной программой с открытым исходным кодом, это минимизирует затраты и повышает удобство разработки.
- Высокая безопасность: Django - безопасный фреймворк, созданный для защиты от уязвимостей и вредоносных атак. Он имеет безопасную систему аутентификации пользователей, которая обеспечивает защиту при управлении паролями и учётными записями пользователей.
- Масштабируемость: современные приложения должны обеспечивать высокую масштабируемость для удовлетворения постоянно увеличивающихся потребностей пользователей. Django может быстро масштабироваться, чтобы поддерживать рост вашей пользовательской базы и выдерживать большой трафик.
- Гибкость: с помощью Django можно создавать самые разные программные продукты, от одностраничных сайтов до сложных приложений.
- ORM: Django позволяет взаимодействовать с бд с помощью объектов Python, упрощая операции с бд и сокращая количество SQL-запросов. 
- Расширяемость: в Django есть система конфигурации, которая позволяет подключать к проекту сторонний код. 
- Панель администратора: Django предоставляет удобную панель администратора для управления содержимым сайта, пользователями и разрешениями, которую можно настроить в соответствии с конкретными потребностями.
- Простое управление: понятная архитектура Django обеспечивает простое управление задачами разработки от начала до финального развёртывания. Стоит подчеркнуть, что он использует Python для работы с файлами, настройками и моделями данных.
Минусы Django:
- Сложность: Django - высокоуровневый фреймворк, который содержит большое количество кода, следовательно, для его обработки понадобится мощный сервер, который будет избыточным для маленьких проектов.
- Множественные запросы: Django не может обрабатывать несколько запросов одновременно. Иными словами чем больше запросов и баз данных, тем ниже будет производительность. Это означает, что, несмотря на возможность быстрого масштабирования, важно следить за общей архитектурой проекта. Иначе это приведет к проблемам со скоростью в будущем. 

Основные этапы и действия, которые происходят при отправке запроса показа страницы в Django:
1.	Человек открывает браузер и вводит URL сайта.
2.	Браузер отправляет запрос на сервер, где расположен веб-сайт.
3.	Сервер получает этот запрос и передает его в Django.
4.	Django получает запрос и смотрит на URL, чтобы понять, какая страница должна быть показана.
5.	Django проверяет файл urls.py в своей структуре проекта, чтобы найти соответствующий URL-шаблон для этой страницы.
6.	Когда Django находит соответствующий URL-шаблон, он вызывает соответствующую функцию представления (view, которую мы описывали выше).
7.	Функция представления обрабатывает запрос и выполняет все необходимые действия, такие как получение данных из бд или выполнение бизнес-логики.
8.	Если данные нужны для отображения страницы, Django может использовать модели данных и базу данных для извлечения этих данных.
9.	Далее Django использует шаблон HTML, который соответствует данной странице.
10.	Django заполняет шаблон данными, которые были получены из бд.
11.	Окончательная HTML-страница готова и возвращается обратно на сервер.
12.	Сервер отправляет эту HTML-страницу обратно в браузер пользователя.
13.	Браузер получает HTML-страницу и начинает ее отображать, пользователю.
Фреймворк Django отлично подходит для разработки веб-сайтов, социальных сетей, блогов и других проектов. Он предоставляет разработчикам мощный и гибкий инструмент для создания безопасных и масштабируемых веб-приложений на Python.

#### 2.2 FastAPI  

FastAPI - современный (был выпущен в 2018 году), высокопроизводительный фреймворк, специально разработанный для создания API на Python.
Особенности FastAPI:
- Встроенная система безопасности: FastAPI включает в себя встроенные механизмы безопасности, к примеру: проверка и очистка пользовательского ввода.
- Масштабируемость: FastAPI - один из немногих фреймворков Python, который можно использовать для создания масштабируемых приложений.
- Автоматическая проверка данных: FastAPI предоставляет механизм автоматической проверки данных, который намного быстрее традиционной ручной проверки, такая проверка гарантирует, что данные актуальны.
- Высокая производительность: FastAPI - современный фреймворк, который обеспечивает производительность наравне с таким многопоточным языком программирования, как Go.
- Поддержка асинхронного кода: FastAPI поддерживает асинхронный код, благодаря ему запросы выполняются независимо друг от друга и могут запускаться параллельно, а это значит, что время выполнения кода значительно сокращается. 

Минусы FastAPI:
- Сообщество: так как FastAPI является относительно новым фреймворком, он имеет не большую поддержку сообщества по сравнению с более старыми аналогами.
- Обучение: хоть и FastAPI не самый сложный фреймворк, потребуется время, чтобы освоить асинхронное программирование и Pydantic для проверки данных.
FastAPI он простой в использовании и в изучении, легко разворачивается, а также дает разработчику преимущество асинхронной обработки запросов. 

#### 2.3 Flask

Flask - бесплатный веб-фреймворк, разработанный в 2010 году, с открытым исходным кодом, написанным на языке Python для простого создания масштабируемых веб-приложений. Некоторые известные организации, которые используют Flask: Reddit, Netflix, Uber и т. д.
Особенности Flask:
- ORM: в отличие от Django, Flask не включает ORM, что позволяет разработчикам выбирать предпочтительную библиотеку для взаимодействия с бд.
- Лёгкость: Flask не зависит от внешних библиотек, что делает его хорошим вариантом для новичков, которые хотят с лёгкостью создавать сложные приложения.
- Минималистичность: Flask предоставляет только основные компоненты для веб-разработки, такие как маршрутизация и шаблоны, позволяя разработчикам добавлять функции с помощью расширений.
- Гибкость: Flask позволяет разработчикам структурировать свои приложения так, как они считают нужным, что делает его подходящим для широкого спектра задач, от небольших проектов до крупномасштабных приложений.
Минусы Flask:
- Однопоточный и синхронный: Flask - фреймворк, в котором каждый процесс может быть запущен только после завершения предыдущего, то есть проявляет синхронное поведение.
- Ориентирован на HTML: создание API не является основной задачей для Flask.
- Безопасность: Использование сторонних модулей может создать угрозу безопасности проекта.

Flask подходит для новичков и позволяет быстро создать веб-приложение, используя только один файл Python. Flask можно использовать при разработке как тренировочных проектов или небольших сайтов, которым не нужен сложный бэкенд.

### 4. Анализ и сравнение работы на Django, Flask и FastAPI
----
Понимание сильных и слабых сторон каждого фреймворка поможет вам сделать правильный выбор для ваших проектов по веб-разработке, поэтому сейчас мы определим в каких ситуациях больше подходят Flask, FastAPI и Django.

#### 4.1 FLASK - FASTAPI

Flask и FastAPI - два главных претендента в мире Python, каждый из которых предлагает что-то уникальное. Независимо от того, работаете ли вы над простым веб-сайтом или мощным API, знание различий между Flask и FastAPI поможет вам принять оптимальное решение для написания проекта.

Проверка достоверности данных

Flask не поддерживает встроенную проверку данных, если вы попытаетесь передать данные не того типа, который требуется, программа просто зависнет. Разработчики могут использовать внешние библиотеки, для проверки данных. С другой стороны FastAPI, обеспечивает встроенную автоматическую проверку данных. FastAPI делает это, поскольку он интегрирован с моделями Pydantic. Эти модели используют аннотации типов Python, которые помогают проверять запросы и ответы API.

Асинхронные задачи

Flask не поддерживает асинхронные задачи, для этого требуются внешние ресурсы. Поэтому добиться высокой производительности без поддержки асинхронного программирования становится сложнее. FastAPI благодаря своей структуре, предназначенной для асинхронного программирования, может обрабатывать большие объемы одновременных запросов и идеально подходит для приложений в формате online (реального времени).

Поддержка сообщества

Flask, будучи более старым фреймворком, имеет большую поддержку сообщества. У него много ресурсов, от учебных пособий и руководств до сторонних расширений и плагинов. FastAPI - относительно новый фреймворк с меньшим сообществом, которое постепенно увеличивается благодаря его высокой производительности и большему объему функций.

В итоге хотел бы сказать, что Flask - микрофреймворк, а FastAPI все же полнофункциональный фреймворк. FastAPI намного лучше, когда речь идёт о скорости, производительности и работы с большим объемом данных. Flask подходит для более простых приложений.

#### 4.2 DJANGO - FLASK

Теперь рассмотрим конкретные особенности Django и Flask, чтобы определить, какой фреймворк лучше подходит в тех или иных ситуациях. Выбор между Django и Flask зависит от различных факторов: от размера и сложности вашего проекта, до ваших знаний и умений работать с фреймворками и опыта вашей команды. 

Безопасность

Flask и Django имеют встроенные аналогичные механизмы безопасности. Так как в Django есть встроенная система для предотвращения распространённых атак, это помогает предотвратить или смягчить последствия, которые позволяют злоумышленнику выполнять скрипты в браузере и выполнять произвольный SQL-код в базе данных. Поскольку приложения Flask намного меньше по размеру, их проще защитить, так как для атаки доступно меньше ресурсов. Но Flask полагается на внешние расширения безопасности, это может означать, что, если само расширение будет скомпрометировано, это приведёт к компрометации веб-приложения Flask.

Поддержка сообщества

Django и Flask имеют достаточно большое сообщество, которое до сих пор выпускает множество учебных руководств и платформ, на которых пользователи делятся решением проблем с которыми сталкиваются. По популярности Django немного опережает Flask. Но у Flask исключительная база поклонников, так как он был выпущен через 5 лет после Django и был признан самым известным веб-фреймворком Python в 2018 году.

Flask станет оптимальным выбором для небольших и несложных веб-приложений с высокой нагрузкой. В свою очередь Django - отличный выбор для больших, сложных и высоконагруженных приложений. Поскольку Flask является модульным и очень гибким, вы контролируете всё, что находится внутри него, и масштабирование может привести к появлению ошибок и сложного кода.  В Django сравнительно легко масштабироваться благодаря полнофункциональным возможностям, которые он предоставляет.

#### 4.3 DJANGO - FASTAPI

Django - фреймворк, включающий в себя большое количество возможностей, тогда как FastAPI - фреймворк, в котором акцентирование идет на простоту использования и производительность. Разберем основные различия между этими 2 инструментами разработки

Гибкость

Django предоставляет стандарт кодирования, и ожидает, что все будет сделано определенным образом по определенным правилам, но FastAPI работает иначе, не ограничивая разработчика определенными правилами, он дает ощущение свободы при написании кода. 

Безопасность

FastAPI имеет несколько подключаемых инструментов по вопросам безопасности. Django же имеет очень хорошую безопасность, чтобы избежать распространенных атак.  Поэтому Django считается более безопасным.

Поддержка сообщества

Django является более распространенным фреймворком среди разработчиком, так как он старше и масштабнее. Но на GitHub FastAPI имеет 79.3 тысяч звезд в то 
время, как Django 81.7 тысяч звезд. Это значит, что сообществу нравятся возможности нового инструмента, и его активно применяют на проектах.

В заключение если приложение должно быть небольшим, но производительным, и при этом простым в разработке и поддержке, то FastAPI - лучший выбор. Но разработка на Django отлично подойдет, если вы хотите создать масштабный продукт, потому что он имеет множество встроенных функций и очень хорошо работает при большой нагрузке.

### 5. Вывод
----

FastAPI несложный процесс, ведь он не накладывает строгих ограничений на архитектуру проекта, а также не имеет предустановленных инструментов для взаимодействия с базой данных и защиты. Плюсом данного фреймворка является разработка посредством асинхронных функций, что делает его крайне быстрым. Ещё один немаловажный плюс - использование декораторов запросов. Данная особенность делает разработку отображений более систематизированной, понятной и удобочитаемой. Название отображает самое явное назначение этого фреймворка: он идеально подходит для разработки различных API. FastAPI - пустой холст для разработчика: на данном фреймворке разработчик сможет реализовать практически любую задумку именно так, как он хочет. Но главным минусом является отсутствие предустановленных библиотек для взаимодействия с базами данных и защиты данных. Несмотря на то, что уже давно выведен набор FastAPI, SQLAlchemy и Alembic, возникают сложности с миграциями и взаимодействием с базами данных.

Разработка на Django подразумевает очень строгие требования к иерархии и архитектуре проекта, но это дает свои плюсы.  За счёт этого разработчик меньше задумывается о подключении тех или иных приложений к проекту, настройки связи с базой данных, а также меньше беспокоится о защите. Минусом же Django является его же плюс: новичку будет достаточно сложно разобраться в том, как правильно взаимодействовать с файлами фреймворка. Если понять, как именно должны располагаться файлы, то разработка на Django становится не сложной и понятной. Благодаря строгости в архитектуре большинство моментов по связи отдельных компонентов программы берёт на себя сам Django.

Flask является чем-то средним между Django и FastAPI. В данном фреймворке изначально также предоставлен достаточно малый набор инструментов для разработки приложения. Также, как у FastAPI, Flask не предъявляет очень строгих требований к архитектуре проекта, что делает его не таким сложным и требовательным к разработчику, как Django. В данном фреймворке, по аналогии с FastAPI, используются декораторы, что делает разработку систематизированной и удобной для восприятия.
